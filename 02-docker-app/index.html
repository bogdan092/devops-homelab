1. Infrastructure Layer (Terraform + KVM)

Terraform acts as the Architect. It reads the blueprint and tells the Hypervisor (KVM) exactly what to build.

graph LR subgraph "Your Workstation (HP Z4)" A[main.tf] -->|terraform apply| B(Terraform Engine) C[cloud_init.cfg] --> B end B -->|libvirt API| D[KVM / QEMU] subgraph "Virtual Reality" D --> E[VM: node-01] D --> F[VM: node-02] end style A fill:#5C4EE5,color:#fff style D fill:#f96,stroke:#333

Commands: terraform plan → terraform apply

2. Application Layer (Docker)

Docker acts as the Shipping Container. It bakes the code into an image so it runs the same way on any machine.

graph TD subgraph "Build Phase" G[Dockerfile] -->|docker build| H[Custom Image] I[index.html] --> H end H -->|docker run| J(Running Container) subgraph "Network Access" K[Browser] -->|Port Mapping| J end style G fill:#2496ed,color:#fff style J fill:#2496ed,color:#fff

Commands: docker build → docker run
